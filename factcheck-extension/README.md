# Factcheck — расширение для браузера

Tech Vision 2026 · Направление: Digital Economy & Future Tech · Зона 13: Медиа-решения

**Один пользователь, одна боль:** активный пользователь соцсетей, который встречает недостоверные утверждения (в тексте или по ссылке) и хочет быстро понять, есть ли в них признаки манипуляции — не дожидаясь, пока кто-то другой это проверит.

## Что это НЕ делает (сознательно)

Инструмент не выносит вердикт "правда/ложь" — ИИ не может это делать надёжно, особенно для свежих событий. Вместо этого он называет конкретные признаки манипулятивной подачи и даёт короткую рекомендацию. Фактчекинг конкретных фактов — roadmap-задача с более сложной инфраструктурой (поиск по проверенным источникам).

## Архитектура

```
extension/            — Chrome-расширение (Manifest V3)
  manifest.json         — конфигурация, разрешения
  background.js          — service worker: делает fetch к бэкенду
  content.js              — плавающая кнопка на любой странице + панель ввода
  overlay.css             — стили

api/check.js          — единственный бэкенд-файл (Vercel serverless function).
                         Содержит всю цепочку: извлечение текста → локальный
                         список паттернов → LLM-классификация → сборка ответа.
                         Не тонкий адаптер поверх стороннего API — своя логика
                         на каждом шаге (см. ниже "Цепочка обработки").
package.json          — зависимости (@supabase/supabase-js, cheerio)
```

Зависимость от Vercel — только неявная (папка `api/` автоматически становится
serverless-функциями без дополнительного `vercel.json`).

## Цепочка обработки (важно для критерия "не пустая обёртка над API")

1. **Извлечение текста** — если на входе ссылка, скачивается HTML и вытаскиваются
   `<p>`-параграфы (`cheerio`); если текст — используется как есть.
2. **Локальный список паттернов** (`LOCAL_PATTERNS` в `api/check.js`, 12 записей) —
   обычное сопоставление по ключевым словам, БЕЗ обращения к ИИ: срочные переводы
   денег, поддельные уведомления госорганов/банков, дропперство, инвестпирамиды,
   цепочные сообщения "перешли всем", паника вокруг ЕНТ и т.д. — паттерны, типичные
   именно для казахстанского контекста (WhatsApp/Telegram-рассылки, местные схемы).
3. **LLM-классификация** (OpenRouter, `openrouter/free`) — получает найденные на
   шаге 2 совпадения как контекст в системном промпте, ищет остальные признаки
   манипуляции (эмоциональное давление, отсутствие источника, абсолютные заявления
   и т.д.), возвращает структурированный JSON.
4. **Сборка ответа** — локальные совпадения помечаются `[локальный паттерн]` в
   начале списка `signals`, отдельно от того, что нашла LLM — на защите видно,
   какая часть результата не зависит от модели.

**Поток данных:** пользователь вставляет текст/ссылку → `content.js` определяет тип
→ `background.js` пересылает на `/api/check` → шаги 1–4 выше → расширение
показывает результат в панели.

## Локальный запуск (для отладки)

```bash
npm install
npx vercel dev
```

`vercel dev` поднимает точную копию продакшен-окружения локально (включая
`api/check.js` как serverless-функцию) — отдельный Express-сервер не нужен.

Понадобится переменная окружения `OPENROUTER_API_KEY` — либо через
`vercel env pull`, либо вручную в `.env.local`:
```
OPENROUTER_API_KEY=ваш_ключ
```
(опционально `SUPABASE_URL` + `SUPABASE_ANON_KEY` для логирования проверок —
без них функция просто пропускает этот шаг, не падает).

## Деплой на Vercel

1. Закоммитьте и запушьте изменения в GitHub.
2. На [vercel.com](https://vercel.com) → Project Settings → Environment Variables
   → добавьте `OPENROUTER_API_KEY` (и опционально `SUPABASE_URL`,
   `SUPABASE_ANON_KEY`).
3. Redeploy — Vercel сам подхватит `api/check.js` без дополнительного `vercel.json`.

**Проверить, что бэкенд реально работает** (не заглушка):
```bash
curl -X POST https://ваш-проект.vercel.app/api/check \
  -H "Content-Type: application/json" \
  -d '{"type":"text","content":"Переведи срочно на карту, не могу говорить"}'
```
В ответе должен быть `signals` с `[локальный паттерн] Срочный перевод денег` —
если вместо этого приходит `"Успешный тест системы"`, деплой ещё не обновился
(проверьте, что старый `api/check.js`-заглушка удалён и файлы из этого архива
закоммичены).

## Установка расширения (для демонстрации)

1. Откройте `chrome://extensions`
2. Включите "Режим разработчика" (правый верхний угол)
3. "Загрузить распакованное расширение" → выберите папку `extension/`
4. На любой странице появится плавающая кнопка 🔍 в правом нижнем углу

## Известные ограничения (честно, для TVEP)

- Не работает поверх других приложений — только поверх веб-страниц (ограничение платформы браузерных расширений).
- Скриншоты/картинки не поддерживаются в MVP — нужна vision-модель, вынесено в roadmap.
- Фактчекинг конкретных утверждений не реализован — только распознавание паттернов манипуляции.
- Извлечение текста по ссылке простое (`<p>`-теги), не работает надёжно на сайтах с JS-рендерингом.
- Список локальных паттернов (`LOCAL_PATTERNS`) — стартовый набор из 12 записей, не исчерпывающий; расширяется по мере CustDev.

## Roadmap (после хакатона)
